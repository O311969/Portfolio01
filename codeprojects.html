VIEWING: PROJECT 1

The Concept: Art in Motion

This project is my debut portfolio, but I wanted to avoid the feel of a standard corporate resume. Instead, I envisioned it as an interactive sketchbook. My goal was to bridge the gap between my traditional art background and my new path in programming, creating a digital space that feels hand crafted and personal.

Hand-Drawn & Hand-Coded

  • Original Illustrations Every visual element from the character mascots to the UI buttons was hand drawn by me to ensure the site is a true reflection of my artistic style.
  • Animated Nodes I replaced standard menus with custom frame-by-frame GIFs of "flowers," creating a discovery based navigation experience.
  • Signature Style This project allowed me to merge my passion for drawing with my interest in digital editing, establishing a unique visual language for my first digital home.

The Developer’s Journey & Learning Curves

Building this was my first experience diving deep into HTML, CSS, and JavaScript. While the drawing phase was where I felt most at home, the technical execution provided a rewarding challenge:

  • The Struggle with Relatives A major hurdle was mastering relative positioning. Trying to make hand-drawn art sit "just right" in a digital layout was a steep learning curve that I am still refining.
  • Pursuing Responsiveness Making a "sketchbook" feel perfect on every screen size is complex. While I am still working toward making the responsiveness perfect, this project represents a massive leap in my understanding of CSS logic.
  • Focus & Depth I implemented a JavaScript system that applies CSS blur filters to the background when an overlay is open, mimicking a camera’s "depth of field" to keep the focus on my art and text.

VIEWING: PROJECT 3

The Moon Landing

This project served as a personal challenge to bridge the gap between hand-drawn digital art and programmatic animation. The goal was to build an immersive experience using only the core web technologies I've mastered, ensuring the logic was as clean as the visual design.

1. Custom Asset Integration (The Moon)

The centerpiece of this project is a hand-drawn moon asset. I used CSS to wrap this image in a "glowing ball" container, utilizing border-radius and box-shadow animations to give the art a digital "pulse."

2. Procedural Animation with JavaScript

I wrote a JavaScript function, createStar(), to handle procedural generation:

  • Dynamic Positioning Calculates screen width and height to place stars randomly.
  • Memory Management Uses setTimeout to remove stars after 5 seconds, preventing lag.

4. Keyframe Orchestration

The atmosphere is driven by four distinct @keyframes sets:

  • glowing: Controls the moon's inner light.
  • fadeInOut: Manages the "Click Me" hint visibility.
  • twinkle & glow: Creates the rhythmic pulsing of the starfield.

3. State Management & Interaction

I handled the transition through a custom openMainPage() function:

  • Scaling Logic Triggers a scale(5) transform for a "zoom" effect.
  • Flag Logic Uses a boolean variable to prevent multiple clicks.
  • Interval Clearing Uses clearInterval to clean up background processes.
Logic: Procedural Star Generation
function createStar() {
    const star = document.createElement('div');
    star.className = 'star';
    star.style.left = `${Math.random() * window.innerWidth}px`;
    star.style.top = `${Math.random() * window.innerHeight}px`;
    document.body.appendChild(star);
    setTimeout(() => { star.remove(); }, 5000);
}

VIEWING: PROJECT 4

Logic & Calculation

This project was my first venture into handling real-time user input. While the design is minimalist, the core challenge was ensuring that the mathematical logic translated correctly from button clicks to the display area.

1. The DOM & Append Logic

I utilized a modular JavaScript function, appendOperation(), to interact with the Document Object Model (DOM). Instead of hardcoding every button, this function dynamically updates the 'resultArea' whenever a user interacts with the keypad.

2. Mathematical Evaluation

For the processing engine, I implemented the eval() method. This allowed the application to interpret strings as mathematical expressions, handling complex order-of-operations automatically.

  • Efficient Syntax By treating the display content as a string, I reduced the need for multiple temporary variables.
  • Instant Feedback The result is calculated and rendered immediately upon pressing the '=' key.

3. UI Design with Table Layouts

I used a semantic HTML Table structure to ensure a perfectly aligned grid. This was a key exercise in mastering structural CSS, specifically focusing on:

  • Hover States Interactive feedback through color shifts (rgb 20 to 30).
  • Visual Hierarchy Using 'highlight' classes for operators to distinguish them from numbers.
JavaScript: Logic & Result Handling
function calcResult() {
    let container = document.getElementById('resultArea');

    container.innerHTML = result;
